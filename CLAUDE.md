## Project Overview

**Journées de l'Innovation Filière Bois** — Site web des Journées de l'Innovation de la Filière Bois, événements régionaux portés par Xylofutur (pôle de compétitivité forêt-bois) et les Fibois régionales.

Full-stack project combining a Strapi v5 headless CMS with an Astro 6 SSR frontend.

**Key Technologies:**
- **Backend**: Strapi v5.40 (Node.js headless CMS, REST API, PostgreSQL)
- **Frontend**: Astro 6 SSR with Node.js adapter, React 19 for interactive components
- **Styling**: Tailwind CSS v4 (CSS-based config via `@tailwindcss/vite`)
- **Package Manager**: pnpm
- **Infrastructure**: Docker Compose with base + overlay pattern, multi-stage Dockerfiles

## Quick Start

```bash
# Start Postgres
make infra-up

# Start Strapi (terminal 1)
cd strapi && pnpm develop
# → http://localhost:1337/admin

# Start web (terminal 2)
cd web && pnpm dev
# → http://localhost:4321
```

On first Strapi start, the bootstrap sets French as the default locale.

## Key Architecture Decisions

### Strapi v5
- API responses are flat: `entity.title`, `entity.image?.url`
- Use `strapi.documents()` (Document Service) for all data operations
- i18n is a core plugin, no separate dependency needed
- API is public (no authentication required)

### Astro SSR
- All pages are server-rendered at request time (`output: "server"`)
- React components hydrate client-side via `client:visible`

### Tailwind v4
- Theme defined in CSS (`src/styles/app.css` with `@theme` block)
- Uses `@tailwindcss/vite` plugin in Astro's Vite config
- Colors: Xylofutur green (#00B194), gold/wood accent (#BA8748), beige surfaces

### Reverse Proxy (Traefik)
- Traefik v3 handles TLS termination with Let's Encrypt ACME
- Single-domain routing: web serves `/`, Strapi serves `/strapi`, MCP serves `/mcp` — all via path prefix + strip middleware
- Disabled in local development (Strapi and web run on the host)
- Configured via Docker labels in the environment overlays

### CI/CD (GitHub Actions)
- `dev.yml`: Builds and pushes Docker images to GHCR on push to main or tags
- `deploy.yml`: Manual workflow for SSH remote deployment with environment selection, version input, optional DB reset
- Secrets managed via GitHub repository secrets, merged into `.env.secrets` at deploy time

### Content Management MCP
- The MCP server (`strapi/mcp/`) runs as a Docker service alongside Strapi and is exposed via Traefik at `/mcp`
- Clients connect their AI tool (Claude, ChatGPT, Gemini, Cursor, etc.) by pointing it at `https://{domain}/mcp` — no local installation needed
- Supports both Streamable HTTP (current MCP spec) and legacy SSE transport for broad tool compatibility
- Authentication uses Strapi's built-in API tokens: clients create a token in Strapi admin and pass it via the `Authorization` header in their MCP config — no server-side secrets needed
- Uses a static config file (`strapi/mcp/content-types.json`) that describes available content types and their fields
- **Maintenance contract**: `content-types.json` MUST be kept in sync with actual Strapi content types. When content types are added, modified, or removed, update this file to match
- AI agents discover the MCP via `/.well-known/mcp.json` (structured), `/llms.txt` (summary), and `/llms-full.txt` (full setup guide) — all served as static files from the Astro frontend
- `strapi/mcp/SETUP_PROMPT.md` is the internal setup guide for agents working within the project
- The MCP server is part of the skeleton and must be maintained with the same care as any production code

### Environment variable split
- `STRAPI_URL` — server-side only (in Astro frontmatter)
- `REACT_STRAPI_URL` — public URL passed as props to React components
- `PUBLIC_*` — available everywhere (Matomo, cookie consent)

### Docker naming convention
- All containers use `jib-` prefix (jib = Journées Innovation Bois)
- Network: `jib-network`
- Images: `jib-strapi`, `jib-web`

## Quality Audits

- `/quality` — **Start here.** Interactive quality audit across 5 categories. Saves reports to `reports/quality/`.
- `/accessibility` — Standalone WCAG 2.1 AA audit (axe-core + jsx-a11y)
- `/seo` — Standalone SEO audit (Lighthouse + meta/OG/structured data)
- `/ecoconception` — Standalone eco-design audit (EcoIndex + RGESN)
- `/gdpr` — Standalone GDPR compliance audit
- `/security` — Standalone web security audit (OWASP, headers, deps, input validation)

All audits require the dev server running (`cd web && pnpm dev`).

## Deployment

- `/setup-deploy` — Interactive wizard to configure deployment: generates secrets, sets up GitHub secrets/variables/environments, creates local `.env` files. Run once before first deployment.
- `/deploy` — Deploy to dev or prod. Runs pre-flight checks, triggers the GitHub Actions pipeline, monitors progress. Designed for non-technical users.
