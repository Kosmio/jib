## Key Architecture Decisions

### Strapi v5
- API responses are flat: `entity.title`, `entity.image?.url`
- Use `strapi.documents()` (Document Service) for all data operations
- i18n is a core plugin, no separate dependency needed
- API is public (no authentication required)

### Astro SSR
- All pages are server-rendered at request time (`output: "server"`)
- React components hydrate client-side via `client:visible`

### Tailwind v4
- Theme defined in CSS (`src/styles/app.css` with `@theme` block)
- Uses `@tailwindcss/vite` plugin in Astro's Vite config
- Colors: Xylofutur green (#00B194), gold/wood accent (#BA8748), beige surfaces

### Environment variable split
- `STRAPI_URL` вЂ” server-side only (in Astro frontmatter)
- `REACT_STRAPI_URL` вЂ” public URL passed as props to React components
- `PUBLIC_*` вЂ” available everywhere (Matomo, cookie consent)

### Docker naming convention
- All containers use `jib-` prefix (jib = JournГ©es Innovation Bois)
- Network: `jib-network`
- Images: `jib-strapi`, `jib-web`

## Quality Audits

- `/quality` вЂ” **Start here.** Interactive quality audit across 5 categories. Saves reports to `reports/quality/`.
- `/accessibility` вЂ” Standalone WCAG 2.1 AA audit (axe-core + jsx-a11y)
- `/seo` вЂ” Standalone SEO audit (Lighthouse + meta/OG/structured data)
- `/ecoconception` вЂ” Standalone eco-design audit (EcoIndex + RGESN)
- `/gdpr` вЂ” Standalone GDPR compliance audit
- `/security` вЂ” Standalone web security audit (OWASP, headers, deps, input validation)

All audits require the dev server running (`cd web && pnpm dev`).

## Deployment

- `/setup-deploy` вЂ” Interactive wizard to configure deployment: generates secrets, sets up GitHub secrets/variables/environments, creates local `.env` files. Run once before first deployment.
- `/deploy` вЂ” Deploy to dev or prod. Runs pre-flight checks, triggers the GitHub Actions pipeline, monitors progress. Designed for non-technical users.
